## Summary

The omni Claude plugin manifests at `plugins/omni/.claude-plugin/plugin.json` and `.claude-plugin/marketplace.json` carry independent semver strings (`2.1.0` and `2.0.0`) while the rest of the omni monorepo follows calver (`2.YYMMDD.N`) synced by `scripts/sync-versions.ts`. This wish extends `sync-versions.ts` to cover both plugin manifests, retrofits them to the current calver, adds a `verify-versions.ts` CI guard to catch future drift, and bundles the change into PR #356 (which currently bumps `plugin.json` to `3.0.0`).

## Scope

### IN
- Extend `scripts/sync-versions.ts` to update:
  - `plugins/omni/.claude-plugin/plugin.json` → top-level `version` field
  - `.claude-plugin/marketplace.json` → every entry in `plugins[]` matching `name === "omni"` → `version` field
- Retrofit the current values of both files to whatever calver the root `package.json` carries when this lands
- Create `scripts/verify-versions.ts` — drift-detection script that compares every tracked version field against the root `package.json` version and exits non-zero on any mismatch
- Add `verify:versions` npm script to root `package.json`
- Wire `bun scripts/verify-versions.ts` into the CI workflow (alongside lint/typecheck)
- Patch PR #356: drop the `3.0.0` plugin.json bump in favor of the calver retrofit + sync-versions extension + verify guard
- Update PR #356 description to reflect the bundled version unification scope

### OUT
- Migrating root or workspace packages from calver to semver
- Marketplace re-publish workflow / npm publish steps
- `audio-decode-shim` (vendored fork — already excluded from sync)
- `version.json` (gitignored build artifact, generated by `generate-version.ts`)
- Pre-commit hook (CI guard is sufficient)
- Tauri/Docker/Cargo (none found in scan)
- Modifying the genie plugin or any other repo

## Decisions

| Decision | Rationale |
|----------|-----------|
| Calver everywhere (`2.YYMMDD.N`) | Smallest change. Matches existing 18 packages. Genie plugin precedent (`4.260401.1`) proves Claude marketplace tolerates calver. User: "version for everything calver" |
| Bundle into PR #356 | Single clean PR. Avoids landing a wrong version on purpose. #356 is already in review and touches the same plugin manifest |
| Extend `sync-versions.ts`, don't fork | One script, one source of truth, one CI step |
| Add `verify-versions.ts` + CI step | Catches future drift the moment a 3rd manifest appears. Cheap insurance — ~30 lines + one CI step |
| Explicit allowlist of tracked fields | Auto-discovery would touch unrelated `version` fields (openapitools, drizzle snapshots). Allowlist is safer |
| Share file allowlist between sync + verify | Both scripts read `scripts/lib/version-fields.ts` so "what files have versions" lives in one place |
| No pre-commit hook | CI catches drift; pre-commit slows every commit for marginal benefit |

## Execution Groups

### Group 1: Sync + Verify + Retrofit + Patch PR #356

**Goal:** Extend the sync tooling to cover both Claude plugin manifests, add a CI drift guard, retrofit current values to calver, and bundle everything into PR #356.

**Deliverables:**

1. **Create `scripts/lib/version-fields.ts`** — shared module that exports the canonical list of tracked version locations:
   - All `package.json` files via the existing walker (root + `packages/*` + `apps/*`, excluding `audio-decode-shim`)
   - `plugins/omni/.claude-plugin/plugin.json` → write to top-level `version`
   - `.claude-plugin/marketplace.json` → write to each entry in `plugins[]` where `name === "omni"`, field `version`
   - Each entry exposes a `read(): string | null` and `write(version: string): boolean` function so callers don't care about file shape

2. **Refactor `scripts/sync-versions.ts`** to consume `version-fields.ts`. Behaviour preserved:
   - Same CLI: `bun scripts/sync-versions.ts <version>`
   - Same format guard: `^\d+\.\d{6}\.\d+$`
   - Same exit codes
   - Output now reports updates for plugin.json and marketplace.json alongside package.json files

3. **Create `scripts/verify-versions.ts`** — reads root `package.json` version, walks every entry in `version-fields.ts`, prints a table of `file → field → actual → status (OK/MISMATCH)`, exits 0 on all-OK, non-zero on any mismatch.
   - Output is human-readable (one line per file)
   - First line of stdout: `Reference version: <calver>`
   - Last line: `OK: N/M files match` or `FAIL: N/M files mismatch`

4. **Wire `verify:versions` into root `package.json`** — add `"verify:versions": "bun scripts/verify-versions.ts"` to the `scripts` block

5. **Wire CI step** — add a `verify:versions` step to `.github/workflows/ci.yml` (the "Quality Gate" job), after the `bun install` step and before the `Build` step. Confirmed to exist at `.github/workflows/ci.yml`

6. **Retrofit plugin manifests** — run `bun scripts/sync-versions.ts <current-root-version>` (read from `package.json`) so both plugin files now carry the calver. Verify with `bun scripts/verify-versions.ts`

7. **Patch PR #356** — checkout `feat/omni-skill-tiers`, amend the plugin.json change so `version` is calver instead of `3.0.0`, also update marketplace.json plugin entry to calver. Update the PR body to mention the bundled version unification work. Force-push (`--force-with-lease`) to update the PR
