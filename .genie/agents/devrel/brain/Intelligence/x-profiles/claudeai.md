## Feature Overlap Analysis

### What Claude Is Building That Genie Already Has
| Claude Feature | Genie Equivalent | Genie Advantage |
|---------------|-----------------|-----------------|
| Auto mode (classifier) | Auto-approve rules engine | Genie's is configurable per-tool, not just safe/risky binary |
| Projects in Cowork | Boards + Projects + Tasks | Genie has full Kanban, stages, dependencies, assignments |
| Multi-file sync (Excel+PPT) | Isolated worktrees per team | Genie's is git-native, zero merge conflicts |
| Computer use | agent-browser integration | Genie delegates to specialized tools, not one monolithic feature |

### What Claude Is Building That Genie Should Leverage
| Claude Feature | How Genie Benefits |
|---------------|-------------------|
| 1M context window | Agents can hold entire repos. Wish context never truncated. |
| Auto mode | Reduces friction in Genie agent sessions |
| Computer use | Could power browser-based testing, deployment verification |
| Interactive charts | Review dashboards, metrics visualization in agent sessions |

### What Claude Is NOT Building (Genie's Moat)
- **Multi-agent orchestration** — Claude is single-agent. Genie coordinates teams.
- **Wish pipeline** — Claude has no brainstorm→wish→work→review→ship flow.
- **Provider agnosticism** — Claude only works with Claude. Genie works with Codex, any provider.
- **Overnight batch execution** — Claude's "Projects" are interactive. Genie's /dream is autonomous.
- **Decision lineage** — Claude's audit is per-session. Genie's is cross-session, cross-agent, queryable.
