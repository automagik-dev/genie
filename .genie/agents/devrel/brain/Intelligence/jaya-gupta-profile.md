## Key Concepts from Jaya

### 1. Data Context vs Decision Context
> "People are conflating data context and decision context.
> Data context answers: what is true right now?
> Decision context answers: why did we do what we did?"

**Four dimensions of the distinction:**
- **Capture:** Data context can be rebuilt retroactively. Decision context only exists as exhaust of real work — miss the moment, lose the trace.
- **Decay:** Data facts last years. Decision reasoning decays fast.
- **Value:** Data is commodity. Decision traces compound — each one makes the next more accurate.
- **Building:** "A better semantic layer is a project. Decision traces are a compounding asset."

### 2. Context Graphs as the New Platform
The thesis: agent orchestration startups sit in the execution path → see all inputs, policies, exceptions, approvals → can capture decision traces → over time this becomes the most valuable asset (the "context graph").

Incumbents can't build this because they're either:
- In the wrong path (read vs write)
- Siloed (can't see cross-system)
- After the fact (ETL, not real-time)

### 3. "Context is the New Code"
> "context is the new code - and right now it's not easily version-controlled, portable, or auditable"

This is a direct call for tools like Genie that make context structural, versioned, and portable.

### 4. Every Startup is Adopting It
> "Can't think of an AI startup I've spoken to in the past 3 months that hasn't had context graph and decision traces in their pitch."

The concept went from paper → mainstream VC pitch deck in 3 months (Dec 2025 → March 2026).
